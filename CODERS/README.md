## What This Folder Is

This is the landing zone for coding agents working with David across repos.

Use it when the task involves implementation, debugging, refactoring, tests, repo cleanup, deployment, or code review.

This folder does not replace project folders. It gives coders a shared operating room before they enter a specific codebase.

## Rules For Coders

- Identify the real repo before editing.
- Check whether there are existing uncommitted changes.
- Do not overwrite work you did not make.
- Keep code in code repos; keep coordination notes here.
- After meaningful coding work, leave a handoff in the relevant project trench folder.
- If working in an online environment, record what was visible and what was not.
- If tests were not run, say so plainly.
