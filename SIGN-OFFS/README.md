# Sign-Offs

tags: #sign-off #handoff #continuity #coordination

---

## Purpose

This folder is for end-of-session sign-offs.

Every meaningful session should leave one sign-off here, even if project-specific trench reports were also updated. The sign-off is the quick human-readable closeout: what happened, what is still open, and where the next agent should begin.

Project trench reports remain the detailed project-local memory. Sign-offs are the cross-project session trail.

## Rule

One sign-off per session.

Use:

```text
SIGN-OFFS/YYYY-MM-DD-agent-short-topic.md
```

Examples:

```text
SIGN-OFFS/2026-07-21-codex-temporal-zenodo.md
SIGN-OFFS/2026-07-21-kimi-lean-bridge.md
SIGN-OFFS/2026-07-21-claude-site-build.md
```

## Required Sections

- Session focus
- What changed
- What was verified
- What was not verified
- Current open loops
- Next agent should start here
- Personal note, if relevant

Copy `template.md` for new sign-offs.
